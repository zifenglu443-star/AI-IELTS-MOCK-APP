# IELTS Mock Lab 项目问题分析报告

**检查日期**: 2026-07-23  
**项目类型**: Electron 桌面应用 + Web应用（雅思模考工具）  
**代码规模**: 约5,820行 JavaScript + 1,831行 CSS

## 一、严重问题 🔴

### 1. 项目结构混乱，存在大量冗余文件

**问题描述**:
- 项目根目录包含3个几乎相同的项目副本文件夹：
  - `IELTS Mock Lab Clean 2026-07-01`
  - `IELTS Mock Lab Clean 2026-07-01 2`
  - `IELTS Mock Lab GitHub Source 2026-07-02 v3`
- 存在多个版本的压缩包（6个zip文件，总计约194MB）
- `ielts-mock-lab-web-demo` 子项目包含完整的node_modules

**影响**:
- 仓库体积膨胀，克隆和同步速度慢
- 容易编辑错误的版本
- 混淆哪个是实际工作版本
- Git 历史记录混乱

**建议**:
```bash
# 删除冗余文件夹和zip文件
rm -rf "IELTS Mock Lab Clean 2026-07-01"
rm -rf "IELTS Mock Lab Clean 2026-07-01 2"
rm -rf "IELTS Mock Lab GitHub Source 2026-07-02 v3"
rm *.zip

# 将 demo 项目移到单独的仓库或使用 git submodule
```

### 2. 缺少错误边界处理

**问题位置**: `app.js` 全局

**问题描述**:
- 没有全局错误捕获机制
- 大量异步操作缺少 try-catch
- IndexedDB 操作失败时可能导致数据丢失

**代码示例**:
```javascript
// 当前代码 - 第3702行
openAssetDb() {
  const request = indexedDB.open(ASSET_DB_NAME, ASSET_DB_VERSION);
  request.onerror = () => console.warn("Asset database could not be opened.", request.error);
  // 没有恢复机制，用户会看到空白界面
}
```

**建议**:
```javascript
// 添加全局错误处理
window.addEventListener('unhandledrejection', (event) => {
  console.error('Unhandled promise rejection:', event.reason);
  showErrorToast('操作失败，请重试');
});

// 为关键操作添加降级方案
async function openAssetDb() {
  try {
    // ... 原有逻辑
  } catch (error) {
    console.error(error);
    // 降级到内存存储
    return useMemoryStorage();
  }
}
```

### 3. 内存泄漏风险

**问题位置**: 
- `app.js` 第1586行：图片裁剪未释放资源
- 全局事件监听器未清理

**问题描述**:
```javascript
// 第1586行 - 警告已经发现问题
console.warn("No source image found for crop", target.imageCrop);
// 但没有清理已创建的 canvas 和 Image 对象
```

**影响**:
- 长时间使用后应用变慢
- 导入大量材料后内存溢出
- 在 Electron 环境中可能导致崩溃

**建议**:
```javascript
// 清理临时对象
function cleanupImageResources(canvas, image) {
  if (canvas) {
    const ctx = canvas.getContext('2d');
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    canvas.width = 0;
    canvas.height = 0;
  }
  if (image) {
    image.src = '';
    image.onload = null;
    image.onerror = null;
  }
}
```

**问题位置**: AI 导入流程

**问题描述**:
- OCR 处理大型 PDF 时没有进度条

**问题位置**: AI 设置对话框

## 五、安全问题 🔒

### 1. API Key 明文存储

**问题位置**: localStorage

**问题描述**:
```javascript
// 第112行
const AI_SETTINGS_STORAGE_KEY = "ielts-mock-ai-settings";
// API Key 直接存在 localStorage，任何脚本都能读取
```

**风险**:
- XSS 攻击可窃取 API Key
- 恶意浏览器扩展可读取

**建议**:
- Electron 环境使用 `safeStorage` API 加密
- Web 环境提示用户不要在公共电脑保存

### 2. 缺少 Content Security Policy

**问题位置**: `index.html`

**问题描述**:
- 没有 CSP 头
- 可能被 XSS 攻击

**建议**:
```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; 
               script-src 'self' 'unsafe-inline'; 
               style-src 'self' 'unsafe-inline'; 
               img-src 'self' data: blob:;">
```

### 3. 未验证外部输入

**问题位置**: PDF 上传、JSON 导入

**问题描述**:
- 用户上传的 PDF 直接处理
- JSON 解析后未充分验证
- 可能导致 prototype pollution

**建议**:
```javascript
function sanitizeJson(obj) {
  // 删除危险属性
  const dangerous = ['__proto__', 'constructor', 'prototype'];
  dangerous.forEach(key => delete obj[key]);
  
  // 递归清理
  Object.values(obj).forEach(val => {
    if (val && typeof val === 'object') {
      sanitizeJson(val);
    }
  });
  
  return obj;
}
```

**问题描述**:
